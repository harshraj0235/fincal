# News Sitemap — Always Include New Content & Submit to Google Search Console

This doc ensures **all news and shorts** are in the sitemap and that the sitemap is **submitted to Google Search Console** for indexing and for **Google News**.

---

## 1. What goes into the news sitemap

The file **`public/sitemap-news.xml`** is generated by the build and includes:

- **Static news pages:** `/news`, `/news/shorts`, `/news/markets`, `/news/business`, `/news/startups`, `/news/economy`, `/news/tech-business`, `/news/business-analysis`
- **Every article in `contentRegistry`:** all entries from `src/cms-content/contentRegistry.ts` as `https://moneycal.in/news/{category}/{slug}` in **Google News format** (`<news:news>`, publication date, title, keywords)
- **Linked articles from shorts:** any **full-story URL** from `public/custom-shorts.json` (shorts you add via the Add Shorts page / publish API) that points to moneycal.in

So in future:

- **New news article** → add it to **contentRegistry** → next build regenerates the sitemap → that article is in **sitemap-news.xml**.
- **New shorts (daily / trend)** → add via **Add Shorts** and publish to **custom-shorts.json** → next build adds their **full-story links** to the news sitemap (and `/news/shorts` is already there).

---

## 2. Regenerating the sitemap after adding news or shorts

After you add or update **news articles** or **shorts**:

1. **Regenerate sitemaps** (includes news from registry + custom shorts):
   ```bash
   npm run generate-sitemaps
   ```
   Or run a full build (which runs the above):
   ```bash
   npm run build
   ```
2. **Deploy** so the updated `public/sitemap-news.xml` is live at `https://moneycal.in/sitemap-news.xml`.
3. **Submit the sitemap in Google Search Console** (see below).

Optional: to generate **only** the news sitemap from the registry (no other sitemaps):

```bash
npm run generate-news-sitemap
```

Then deploy and submit as above.

---

## 3. Submitting the news sitemap to Google Search Console (for indexing)

1. Open **Google Search Console**: [https://search.google.com/search-console](https://search.google.com/search-console)
2. Select the property for **moneycal.in** (or your live domain).
3. In the left menu go to **Sitemaps** (under “Indexing” or “Indexing” → “Sitemaps”).
4. Under **Add a new sitemap** enter:
   ```text
   sitemap-news.xml
   ```
   (Only the filename; GSC uses the property’s domain.)
5. Click **Submit**.

Google will then use this sitemap for **discovery and indexing**. For **Google News**, the same sitemap is used because it is in **Google News sitemap format** (with `<news:news>` and publication date).

---

## 4. Using this sitemap for news

- **sitemap-news.xml** is in **Google News sitemap format** (`xmlns:news="http://www.google.com/schemas/sitemap-news/0.9"`).
- Each article URL has:
  - `<news:publication>` (name, language)
  - `<news:publication_date>` (ISO 8601)
  - `<news:title>`
  - `<news:keywords>`
- So this sitemap is the one to **submit for both general indexing and for news**. No need to submit a different sitemap for “news” in GSC; use **sitemap-news.xml** for both.

## 5. Checklist for future news/shorts

| Step | Action |
|------|--------|
| 1 | Add new article to **contentRegistry** and/or add shorts (and publish to **custom-shorts.json** if applicable). |
| 2 | Run **`npm run generate-sitemaps`** (or **`npm run build`**) and **deploy**. |
| 3 | In **Google Search Console** → **Sitemaps**, submit **sitemap-news.xml** if not already submitted (resubmit after updates if you want to prompt re-crawl). |
| 4 | New URLs will be picked up from the sitemap for indexing and for news. |

## 6. Where it’s defined in the project

- **Sitemap generation:** `scripts/generate-sitemaps-production.cjs`  
  - Builds news URLs from: contentRegistry + static news pages + **custom-shorts.json** full-story links.
- **News-only sitemap script:** `scripts/generate-news-sitemap-from-registry.cjs`
- **Robots.txt** already lists: `Sitemap: https://moneycal.in/sitemap-news.xml`
- **Add Shorts / publish:** see `docs/ADDING-DAILY-SHORTS-GOOGLE-DISCOVER.md` and `docs/SHORTS-PUBLISH-API-SETUP.md`

Keeping this workflow ensures **every new news article and shorts-linked story** is in the sitemap and submitted to Google Search Console for indexing and for news.
